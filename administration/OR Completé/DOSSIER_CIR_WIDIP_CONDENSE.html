1. Contexte et objectifs

WIDIP gere le support IT pour un portefeuille de PME. Le projet vise a resoudre 4 defis : perte de connaissances, traitement reactif, taches repetitives, risque d'erreur sur actions critiques.

Preuves d'incertitude scientifique

  • OR2 : L'auto-evaluation de confiance par LLM a echoue (scores > 85% meme sur erreurs)
  • OR3 : La synchronisation par polling a echoue, necessitant le mecanisme de bypass
  • Pivot N8N → Python : L'approche low-code s'est averee inadaptee pour les cas complexes

4. Verdict d'eligibilite

Points forts

  • Problemes de recherche legitimes (3 OR)
  • 11 verrous techniques documentes
  • Echecs d'hypotheses demontrant l'incertitude
  • OR3 en production avec metriques
  • Pivot N8N → Python = demarche experimentale

A renforcer

  • Etat de l'art formel pour OR1 et OR2
  • Journal de bord R&D structure
  • Metriques quantifiees pour OR1/OR2
  • Matrice de confusion complete (OR2)

Conclusion : Eligibilite POSITIVE sous conditions

Le projet presente un potentiel CIR significatif sur les 3 operations de recherche. Les verrous techniques sont reels et les incertitudes documentables. L'eligibilite est conditionnee a la production de la documentation R&D manquante pour OR1 et OR2.
