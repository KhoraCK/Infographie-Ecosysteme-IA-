1. Vue d'ensemble

1.1. Objectif du workflow

L'ENRICHISSEUR est un workflow batch qui analyse les tickets résolus pour en extraire des procédures réutilisables. Il transforme l'expérience quotidienne des techniciens en connaissances capitalisées dans la base RAG.

Mission principale

"Transformer chaque résolution de ticket en connaissance réutilisable pour améliorer continuellement les capacités de la plateforme"

Fonctions clés

Fonction Description
Collecte Récupération des tickets clôturés et bien documentés
Analyse Extraction des étapes de résolution via LLM
Détection patterns Identification des résolutions récurrentes
Génération Rédaction automatique de procédures structurées
Validation Soumission pour approbation humaine (Safeguard)
Publication Injection dans le RAG après validation

2. Spécifications fonctionnelles

2.1. Sources de données

L'Enrichisseur analyse trois types de sources pour générer des procédures :

Source Description Priorité
Tickets clôturés Tickets GLPI fermés avec suivi détaillé de la résolution Haute
Tickets flaggés Tickets que l'Assistant Ticket n'a pas pu résoudre, maintenant résolus par un technicien Très haute
Conversations Agent Sessions WIBOT Agent complexes ayant abouti à une résolution documentée Moyenne

2.2. Critères de sélection

Tous les tickets ne sont pas candidats à la génération de procédure. L'Enrichisseur applique des critères stricts.

Critères d'inclusion

Critère Condition Justification
Documentation Suivi > 100 caractères Assez de détails pour extraire une procédure
Résolution Statut = Clôturé/Résolu Solution effective et validée
Ancienneté Clôturé dans les dernières 24-168h Information fraîche et pertinente
Catégorie Catégorie technique supportée Dans le périmètre d'automatisation

Critères d'exclusion

Critère Raison
Suivi trop court (< 50 car.) Impossible d'extraire une procédure utile
Ticket annulé Pas de résolution réelle
Ticket doublon Référence un autre ticket
Données confidentielles détectées Mots de passe, données personnelles sensibles
Procédure déjà existante (RAG match > 90%) Éviter les doublons

2.3. Processus d'analyse

1
Collecte des tickets candidats
Requête GLPI avec filtres de sélection (voir 2.2)
2
Enrichissement contexte
Récupération de l'historique complet, des pièces jointes, des commentaires
3
Vérification doublon RAG
Recherche de procédures similaires existantes (seuil 90%)
4
Analyse LLM
Extraction des étapes de résolution, identification du pattern
5
Score de qualité
Évaluation de la qualité de la procédure générée (0-100)
6
Décision
Si score > 70 : soumettre pour validation. Sinon : archiver pour review manuel

2.4. Génération de procédures

À partir de l'analyse, l'Enrichisseur génère une procédure structurée prête à être intégrée au RAG.

Structure d'une procédure

PROC-AD-015 : Résolution problème synchronisation OneDrive

Catégorie : Office 365 / OneDrive

Niveau : N1 (Technicien)

Temps estimé : 15 minutes

Source : Tickets #4521, #4589, #4612 (pattern récurrent)

SYMPTÔMES

  • OneDrive affiche "Synchronisation en pause"
  • Icône OneDrive avec point d'exclamation rouge
  • Fichiers non synchronisés depuis plusieurs heures

PRÉREQUIS

  • Accès au poste de l'utilisateur (TeamViewer/Anydesk)
  • Droits administrateur local

PROCÉDURE

  1. Fermer l'application OneDrive (clic droit icône > Quitter)
  2. Ouvrir le Gestionnaire des tâches, vérifier qu'aucun processus OneDrive ne tourne
  3. Supprimer le cache : %localappdata%\Microsoft\OneDrive\
  4. Relancer OneDrive depuis le menu Démarrer
  5. Se reconnecter avec les identifiants O365 de l'utilisateur
  6. Attendre la fin de la synchronisation initiale

VÉRIFICATION

  • Icône OneDrive verte (synchronisation OK)
  • Créer un fichier test et vérifier qu'il apparaît sur OneDrive web

Types de procédures générées

Type Source Exemple
Standard Tickets simples récurrents Reset mot de passe, déblocage compte
Avancée Tickets flaggés résolus par tech Diagnostic VPN, migration boîte mail
R&D Conversations Agent complexes Intégration nouveau service, architecture

Options de validation

Action Description Résultat
Approuver Procédure correcte, prête à être publiée Publication RAG
Modifier Procédure nécessite des ajustements (édition inline) Édition puis approbation

Fusionner Procédure similaire existe, enrichir l'existante Mise à jour RAG
Double stockage (optionnel) : Les procédures validées peuvent également être publiées dans la base de connaissances GLPI pour être accessibles depuis l'interface de ticketing.
